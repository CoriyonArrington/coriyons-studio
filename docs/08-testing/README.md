# 🧪 08-testing – Testing Overview – Coriyon’s Studio

The **08-testing** folder defines the testing practices, tools, and guidelines to ensure code quality and reliability across Coriyon’s Studio’s codebase.

These docs cover strategy, unit tests, integration tests, end-to-end flows, mocking patterns, and coverage requirements.

---

## ✅ Who This Is For

* **Developers** writing or reviewing tests  
* **QA engineers** designing test plans  
* **DevOps** maintaining CI pipelines and test infrastructure  

---

## 📁 Folder Structure

- `testing-strategy.md`      – Overall approach, objectives, and metrics  
- `unit-testing.md`          – Vitest unit tests for components and utils  
- `integration-testing.md`   – Testing Next.js API routes and Supabase interactions  
- `end-to-end-testing.md`    – Playwright end-to-end test guidelines  
- `mocking-and-fixtures.md`  – Patterns for fixtures, factories, and mocks  
- `test-coverage.md`         – Coverage thresholds, reporting, and badges
